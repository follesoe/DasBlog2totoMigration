<p><a href="http://larswilhelmsen.com/2008/10/14/pimp-my-visual-studio-2008/" target="_blank">Lars Wilhelmsen</a> started this meme and he is curious about what addins and customizations for Visual Studio 2008 other fellow developers use in their daily work. Lars actually uses lots of useful tools that I’ve never heard of, so it was quite useful for me to read his list.</p>  <p>In my daily practice as a software craftsman I use a lot of tools, but very few of them integrate with Visual Studio. Therefore I would like to raise another question, and that is; what other tools beside Visual Studio 2008 do you have in your tool belt when developing applications based on the .NET platform?</p>  <p align="center"><a href="http://blog.goeran.no/content/binary/WindowsLiveWriter/PimpmyVisualStudio2008anddevelopertoolbe_9B5B/iStock_000005579883Small_2.jpg"><img style="border-width: 0px;" alt="iStock_000005579883Small" src="http://blog.goeran.no/content/binary/WindowsLiveWriter/PimpmyVisualStudio2008anddevelopertoolbe_9B5B/iStock_000005579883Small_thumb.jpg" border="0" width="200" height="300"></a> </p>  <p>Here is my list of addins and customization to Visual Studio 2008:</p>  <h3>Addins</h3>  <ul>   <li><a href="http://www.testdriven.net/">TestDriven.NET</a> – A test-runner that supports various sets of testing frameworks; NUnit, MBUnit, XUnit etc. </li>    <li><a href="http://ankhsvn.open.collab.net/" target="_blank">AnkhSvn</a> – Subversion Source Control into Visual Studio </li>    <li>Theme: <a href="http://winterdom.com/weblog/2008/03/23/DistantShoresAVisualStudioColorScheme.aspx">Distant Shores</a> </li>    <li>Font: Consolas 15pt </li> </ul>  <h3>Templates and snippets</h3>  <ul>   <li><a href="http://www.codeplex.com/CIMonitor/SourceControl/FileView.aspx?itemId=129102&amp;changeSetId=3402">PresentationModel snippet</a> – A snippet I originally created for personal use. I use it when I’m developing WPF application and use the Presentation Model (Mode-View-ViewModel) design pattern. The snippet generate properties </li> </ul>  <h3>Shortcuts</h3>  <ul>   <li>Alt + shift + F10 – Activate smart tag. I use it to import namespaces and generate implementation of interfaces that my class implements </li>    <li>Ctrl + K, K – Bookmark a line of in source file </li>    <li>Ctrl + K, N – Moves to the nest bookmark in document </li>    <li>Ctrl + K, P – Moves to the previous bookmark </li>    <li>Ctrl + B, C – Clear all bookmarks in file </li>    <li>Ctrl + J – IntelliSense </li>    <li>Shift + Alt + Enter – full screen </li>    <li>Ctrl + E, C – Add comment </li>    <li>Ctrl + E, U – Remove comment </li>    <li>Ctrl + Shift + B – Build solution </li> </ul>  <h3>Other tools (developer tool belt)</h3>  <ul>   <li><a href="http://www.codeplex.com/xunit" target="_blank">XUnit</a> – Unit testing framework </li>    <li><a href="http://www.nunit.org/" target="_blank">NUnit</a> – Another unit testing framework </li>    <li><a href="http://code.google.com/p/moq/" target="_blank">Moq</a> – Mocking framework </li>    <li><a href="http://ayende.com/projects/rhino-mocks.aspx" target="_blank">RhinoMocks</a>- Another mocking framework </li>    <li><a href="http://boo.codehaus.org/" target="_blank">Boo</a> – One of my new favorite languages on the .NET platform. Why? Syntax extensions with macros. You can actually extend this language with your own syntax </li>    <li><a href="http://specter.sourceforge.net/" target="_blank">Specter</a> – Unit testing framework implemented in Boo. What’s nice about Specter is that it’s based on the BDD ideology and has a nice DSL for writing those test cases. I also use it for writing tests for code implemented in C# and other .NET languages </li>    <li><a href="http://sourceforge.net/projects/sharpdevelop/" target="_blank">SharpDevelop</a> – Open Source IDE which I use when writing Boo code </li>    <li><a href="http://wix.sourceforge.net/" target="_blank">WiX</a> (Windows Installer Xml) – A neat tool for building Windows Installation packages. My friend Jon Arild has some really good posts about this tool </li>    <li><a href="http://www.finalbuilder.com/finalbuilder.aspx" target="_blank">FinalBuilder</a> – Use this tool for Build Automation and release management. What’s nice about this tool is that it represents the build process in a Visual manner. This certainly lowers the <a href="http://www.finalbuilder.com/blogs.aspx?EntryID=249" target="_blank">bus factor</a> </li>    <li><a href="http://www.finalbuilder.com/automise.aspx" target="_blank">Automise</a> – Visual Batch scripting tool. With this tool it’s really easy to automize manual procedures </li>    <li><a href="http://confluence.public.thoughtworks.org/display/CCNET/Welcome+to+CruiseControl.NET" target="_blank">CruiseControl.NET</a> – Continuous Integration (my favorite for years!) </li>    <li><a href="http://msdn.microsoft.com/en-us/library/0k6kkbsd.aspx" target="_blank">MSBuild</a> – The main build tool for the .NET platform. Did you know, when compiling your solution in Visual Studio, you are actually starting an MSBuild script? I use MSBuild to script build tasks like compile and unit testing </li>    <li><a href="http://www.red-gate.com/products/reflector/" target="_blank">Reflector</a> - If you need to have a look at or reverse some compiled .NET applications (MSIL), this is the tool </li>    <li><a href="http://tortoisesvn.tigris.org/" target="_blank">Tortoise</a> – Client for Subversion which integrate nicely into your Operating System shell </li>    <li><a href="http://www.microsoft.com/expression/products/Overview.aspx?key=blend" target="_blank">Blend</a> – Use Blend for creating GUI (XAML) for Silverlight and WPF </li>    <li><a href="http://www.mikeswanson.com/XAMLExport/" target="_blank">XAMLExport</a> – Illustrator to XAML export </li>    <li>XAML Debuggers (<a href="http://www.codeproject.com/KB/macros/MoleForVisualStudioEdit.aspx" target="_blank">Mole</a>, <a href="http://www.codeproject.com/KB/WPF/WoodstockForWPF.aspx" target="_blank">Woodstock</a>) – Use them for XAML debugging </li>    <li><a href="http://www.jQuery.com" target="_blank">jQuery</a> </li> </ul>  <p>Let me emphasize that I’m using lots of tools, and I’ve probably forgot some of them, but this is the tools that immediately came into my mind.</p>  <p>There is no good reason for me not to use Visual Studio IDE extensions like ReSharper or CodeRush. I’ve several times started to use ReShaper, but there has been something missing every time. For instance, the last time I took it for a test run they didn’t support LINQ, which they do now. I have actually decided to install ReSharper after I’ve published this post, and then take it out for a new test run. </p>  <p>I’m also interested to hear what other fellow developers use for Visual Studio 2008 and other tools they use as a software craftsman.
